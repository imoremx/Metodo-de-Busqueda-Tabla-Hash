Proyecto de Búsqueda mediante Tabla HashEste proyecto implementa un sistema de almacenamiento y búsqueda eficiente utilizando una Tabla Hash con Encadenamiento para gestionar un conjunto de 50,000 números enteros.📈 Análisis de ComplejidadEl rendimiento de este programa se divide en dos fases:1. Preparación de los datos (Pre-procesamiento)Ordenamiento: Se utiliza el algoritmo interno de Python (Timsort). Su complejidad es O(n log n). Aunque el hash no requiere ordenamiento, este paso se realiza para cumplir con los requisitos de organización del archivo de salida.Construcción de la Tabla: Insertar los $n$ elementos en la tabla hash toma O(n), ya que cada inserción individual es una operación constante.2. Búsqueda (Operación Principal)Caso Promedio: O(1). Gracias a la función hash, el acceso a la "cubeta" es directo. Si el factor de carga es bajo (pocos elementos por cubeta), encontrar el número es casi instantáneo.Peor Caso: O(n). Esto ocurriría solo si todos los números generaran la misma clave hash y cayeran en la misma cubeta, convirtiendo la búsqueda en una lista lineal.
